# Code Editors 🖥️

**Code editor** ek aisi jagah hai jahan programmers apna saara kaam aur time spend karte hain code likhte hue. Basically ye ek workshop ki tarah hai jahan coder apna har ek kaam karta hai — yahan programmers apna code likhte hain, edit karte hain ya manage karte hain.

Do main types hote hain code editors ke: **IDEs** aur **lightweight editors**. Kaafi log dono mein se ek ek tool use karte hain.

---

## IDE (Integrated Development Environment)

**IDE** ek powerful tool hai jo project level par kaam mein aata hai. Ye weight mein heavy hote hain (matlab inki storage size zyada hoti hai). Isme bohot saare features hote hain jaise autocompletion, version management system (Git) integration, aur testing — ye sab features ek saath milte hain.

IDE sirf ek editor nahi hota — ye ek full-scale "development environment" hota hai. Ye project ke saare files load karta hai, files ke beech navigation allow karta hai, aur poore project ke basis par autocompletion provide karta hai (sirf open file ke basis par nahi).

### IDE Options

- **Visual Studio Code** — cross-platform, free
- **WebStorm** — cross-platform, paid (lekin trial period hoti hai)

> **Note:** Windows par ek aur option hai — "Visual Studio" (jo "Visual Studio Code" se bilkul alag hai). Ye ek paid, Windows-only editor hai jo mainly .NET platform ke liye suited hai, lekin JavaScript ke liye bhi achha hai. Iska ek free version bhi hai: Visual Studio Community.

---

## Lightweight Editors

**Lightweight editors** utne powerful nahi hote jitne IDEs, lekin ye bohot simple, fast aur elegant hote hain. Inка use mainly kisi single file ko instantly open ya edit karne ke liye kiya jaata hai.

IDEs aur lightweight editors mein main farq ye hai ki IDE project-level par kaam karta hai — start par bahut zyada data load karta hai, project structure analyze karta hai, etc. Lightweight editor sirf tab bahut faster hota hai jab hume sirf ek hi file chahiye hoti hai.

Practice mein, lightweight editors mein bhi directory-level syntax analyzers aur autocompleters jaise bahut saare plugins ho sakte hain — isliye dono ke beech koi strict border nahi hota.

### Lightweight Editor Options

- **Sublime Text** — cross-platform, shareware
- **Notepad++** — Windows, free
- **Vim** aur **Emacs** — ye bhi cool hain agar aap inhe use karna jaante ho

---

## Editor Kaun Sa Choose Karein?

Editor ka choice, kisi bhi aur tool ki tarah, individual hota hai aur aapke projects, habits, aur personal preferences par depend karta hai. Upar listed editors woh hain jo widely use kiye jaate hain aur developers mein popular hain.

> **Recommendation:** Agar mostly frontend develop kar rahe ho to **Visual Studio Code** best choice hai.
